## Agent Flow Implemented

The committee runs ten deterministic specialist agents in dependency order via
`runAgents` — `technical`, `pattern`, `regime`, `order_flow`, `catalyst`, `position`,
`memory`, `bull_case`, `bear_case`, `risk_critic` — then `synthesize` produces a single
dashboard-safe read. Verified by the new per-agent tests:

- Honest degradation: `technical`/`pattern` go `DEGRADED` with `confidence 0` when price/
  VWAP or triggers are unavailable; `regime` is always honestly `DEGRADED` (never invents
  a regime label).
- No invented data: `order_flow`, `catalyst`, `memory` return `UNAVAILABLE` with empty
  supporting factors and `confidence 0`.
- `position` uses position-safe, non-directive language (no buy/sell/enter/exit imperatives),
  both flat and in-position.
- `bull_case`/`bear_case` introduce no facts beyond the provided subreads (plus
  structural gate/block/R:R evidence for the bear case).
- `risk_critic` returns `PASS` on a clean flat event, `BLOCK` on an L5 hard-blocked event,
  and `WARN` on weak validation, bad spread, thin reward/risk, and low trigger credibility.
- Every read of every fixture passes `validateAgentRead` (schema-valid) and
  `scanForbiddenDeep` (no forbidden language).

## Replay Mode Implemented

Fixture-backed deterministic replay (`copilot-core/replay.ts`) replays bars `[0..step]`
through the same pipeline as live reads and never executes, simulates, routes, or
paper-trades. Verified end-to-end: `GET /api/copilot/replay/session` returns session
metadata (80 steps, `dataSource: fixture_replay`) and `GET /api/copilot/replay/explain`
returns a full committee read at a step. The transport (play/pause/step/speed/stop),
replay-load, REPLAY-tagged journaling, and live/replay separation are covered by the
pre-existing `replay.test` and `use-replay-store.test` suites (matrix items 21, 22, 24, 25).

## Strategy Lab Implemented

The Strategy Lab separates the primary edge from entry refinement, and hand-typed
credibility cannot promote folklore patterns (ICT / FVG / liquidity sweep) into a proven
edge. Covered by the pre-existing `strategyLab.test` and `edgeScoreboard.test` suites
(matrix items 26, 27). No changes in this task.

- **Automated scan** (`safetyScan.test.ts`): walks `lib/` + `artifacts/` (excluding

## L5 Block Enforcement

## Known Limitations

- The api-server vitest harness forces the keyless **deterministic** committee path
  (matrix items 13/18/23) by setting `COPILOT_LLM_PROVIDER=none`, so its assertions never
  depend on a live LLM call. The live-provider (OpenAI) path is verified separately by the
  e2e checks above against the running server.
- The automated safety scan matches a curated token/SDK list (snake_case, camelCase,
  spaced, and broker-name forms). It is intentionally conservative; genuinely novel
  obfuscated phrasings would require adding tokens to the list.
- Replay and "live" reads use fixture-backed data and the keyless Yahoo delayed feed; real
  paid historical feeds remain out of scope (adapter hooks only), consistent with the spec.
